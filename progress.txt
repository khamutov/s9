## Codebase Patterns
- Frontend CSS uses CSS Modules for components + global styles in `src/styles/` (tokens.css, reset.css, typography.css, globals.css)
- Design token names use short form without `color-` prefix: `--bg-root`, `--accent`, `--text-primary`
- Three font families: `--font-display` (Bricolage Grotesque), `--font-body` (Instrument Sans), `--font-mono` (Martian Mono)
- Layout components live in `src/components/layout/` with CSS Modules
- React context + hooks must be split into separate files for react-refresh compatibility: context definition in .tsx, state/types in .ts, hook in .ts
- NavLink active state for query-string routes needs custom location matching (useLocation + manual comparison)
- Pre-existing failures: backend clippy warns on dead_code in references.rs; frontend has no test files yet (vitest exits 1); prettier warns on 5 pre-existing files
- Page components go in `features/<domain>/` with `*Page.tsx` naming; lazy-loaded via React.lazy() in router.tsx
- Helper components used in router.tsx must be in separate files (react-refresh rule: files exporting non-components can't contain component definitions)

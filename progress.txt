## Codebase Patterns
- Frontend CSS uses CSS Modules for components + global styles in `src/styles/` (tokens.css, reset.css, typography.css, globals.css)
- Design token names use short form without `color-` prefix: `--bg-root`, `--accent`, `--text-primary`
- Three font families: `--font-display` (Bricolage Grotesque), `--font-body` (Instrument Sans), `--font-mono` (Martian Mono)
- Layout components live in `src/components/layout/` with CSS Modules
- React context + hooks must be split into separate files for react-refresh compatibility: context definition in .tsx, state/types in .ts, hook in .ts
- NavLink active state for query-string routes needs custom location matching (useLocation + manual comparison)
- Pre-existing failures: backend clippy warns on dead_code in references.rs; frontend has no test files yet (vitest exits 1); prettier warns on 5 pre-existing files
- Page components go in `features/<domain>/` with `*Page.tsx` naming; lazy-loaded via React.lazy() in router.tsx
- Helper components used in router.tsx must be in separate files (react-refresh rule: files exporting non-components can't contain component definitions)
- usePageHeader requires `breadcrumb: string[]` — use `[]` for top-level pages, `['Parent']` for sub-pages
- Auth: AuthProvider wraps RouterProvider in main.tsx; AuthGuard wraps RootLayout in router.tsx; useAuth() for auth state anywhere
- Login page is outside RootLayout (no sidebar/header); does NOT use usePageHeader
- Shared UI components (StatusBadge, PriorityBadge, UserPill) live in `src/components/` with CSS Modules
- TanStack Query hooks go in `features/<domain>/use*.ts`; queryKey convention: `['tickets', params]`
- E2E tests: CSS Module class names are hashed, so use role/text locators not CSS selectors; use `{ exact: true }` for ambiguous text matches
- E2E tests: "New"/"In Progress" etc. appear in both status badges and summary bar — use `.first()` or count assertions to avoid strict mode violations
- E2E tests: PageHeader renders an h1 with the title, so detail pages have 2 h1s — use `.first()` or `{ name: '...' }` to disambiguate
- Shared UI components: TypeBadge (bug/feature/task/improvement) added in `src/components/TypeBadge.tsx`
- Ticket detail: description is comment #0, activity is comments #1+; fetch separately via useComments hook
- InlineSelect/InlineText are reusable components in `src/components/` for click-to-edit fields; use `renderValue`/`renderOption` for custom display
- useUpdateTicket mutation hook uses optimistic updates: immediately sets cache, rolls back on error, invalidates on settle
- E2E mockApi fixture supports .patch() method for mocking PATCH endpoints
- Strict React linting: don't use setState in effects (use render-phase sync or startEditing callback), don't access refs during render, jsx-a11y rules not available (use `role="presentation"` for backdrop instead of eslint-disable)
- MarkdownRenderer: uses react-markdown + remark-gfm + custom remarkS9References plugin; lives in `src/components/`
- react-markdown custom URL schemes (e.g. `mention:`) need a custom `urlTransform` prop to bypass the default sanitizer
- Mentions in remark plugin: use link nodes with `mention:` scheme prefix, then render as `<span>` in custom MdLink component (react-markdown doesn't support custom node types well)
- Internal links (starting with `/`) use React Router `<Link>`, hash links use plain `<a>`, external links get `target="_blank"`
- When MarkdownRenderer is used inside a component, tests need `<MemoryRouter>` wrapper
- MarkdownEditor is a named export (`{ MarkdownEditor }`), not a default export
- E2E: "Comment" submit button conflicts with "Edit comment #N" aria-labels — use `{ exact: true }` for the submit button locator
- Comment mutations: useCreateComment/useEditComment/useDeleteComment invalidate `['tickets', ticketId, 'comments']` query key
- Strict lint: cannot call setState in useMemo or useEffect. Derive initial state during render using `const effective = userState ?? computedDefault` pattern
- Import `react-router` (not `react-router-dom`) for Link, useNavigate, MemoryRouter, etc.
- E2E: when text appears in both tree and detail panel, use `.first()` on locators to avoid strict mode violations
- Admin pages: check `useAuth().user?.role !== 'admin'` and show access denied fallback
- Modal pattern: heading + submit button with same text — use `getByRole('heading', { name })` or `getByRole('button', { name })` to disambiguate in tests
- E2E admin tests: sidebar nav has "Components"/"Settings" links — use card descriptions or `.first()` to avoid strict mode violations
